### 选项 A：多选批量操作 (Multi-select Mode) —— **最推荐**

**交互方向**。目前只能杀一个，或者用 `/pkill` 盲杀。

* **痛点**：有时候我想杀掉这 3 个 `node` 进程，但不想杀掉另外 2 个。`/pkill node` 会误杀，逐个杀又太慢。
* **功能**：
* 在列表中按 `Space`（空格）键，可以**勾选/取消勾选**当前行。
* 被选中的行前面显示 `[x]` 或高亮颜色。
* 状态栏显示 `3 processes selected`。
* 按 `X` 时，弹窗提示 `Kill 3 processes?`，确认后一次性带走。


* **难度**：中等（需要修改 `ListView` 的数据模型，维护一个 `selected` 集合，并自定义 List 的渲染逻辑）。

### 选项 B：深度检视 (Deep Inspection)

**数据方向**。现在的详情页只有 CPU/Mem 和命令行。

* **痛点**：调试程序时，我常需要知道“这个进程占用了哪个端口？”（虽然列表有但详情页可以更详细）或者“它的环境变量 `GOPATH` 对不对？”、“它打开了哪个日志文件？”。
* **功能**：
* 改造 `DetailView`，支持 **Tab 切换**。
* Tab 1: **概览** (Overview) - 现在的波形图界面。
* Tab 2: **环境变量** (Env) - 显示 `KEY=VALUE` 列表。
* Tab 3: **打开文件** (Open Files) - 列出该进程持有的文件句柄和网络连接详情。


* **难度**：中等偏高（需要调用 `gopsutil` 的更多 API，处理 Tab 切换的 UI 逻辑）。

### 方向一：网络黑科技 —— 局域网“闪传”工具 (Local AirDrop)

**口味：** 麻辣（并发、网络协议、原生 Socket）
**描述：** 很多时候我们只是想把电脑上的文件发给同一 Wi-Fi 下的手机或另一台电脑，但不想登录微信或网盘。
**要做什么：**
写一个 CLI 工具，运行后启动一个 HTTP 服务，自动获取本机 IP 并生成一个二维码（在终端打印）。手机扫码就能下载电脑指定的文件，或者上传文件给电脑。

* **技术点：** `net/http` 标准库, `goroutine` (同时处理多人下载), IP 地址发现。
* **进阶玩法：** 利用 UDP 广播自动发现局域网内其他运行该软件的设备，实现无服务器的点对点传输。

### 方向二：中间人攻击/流量分析 —— 简易 HTTP 代理 (Mini Proxy)

**口味：** 咸鲜（协议分析、HTTP 头处理、数据劫持）
**描述：** 既然你对网络工具（Clash类）有接触，不如自己写一个简单的 HTTP/HTTPS 代理服务器。
**要做什么：**
程序运行在 `localhost:8080`，把浏览器的代理设过去。所有的请求都会经过你的程序。你可以打印出所有的请求 URL，甚至修改网页返回的内容（比如把所有网页图片换成滑稽脸）。

* **技术点：** `net` 包, `io.Copy` (数据管道), HTTPS 的 `CONNECT` 方法处理。
* **有趣之处：** 你会深刻理解 HTTP 协议和 TCP 隧道。

### 方向三：复古游戏模拟器 —— Chip-8 Emulator

**口味：** 苦咖啡（硬核、位运算、CPU指令集）
**描述：** 这是程序员的“浪漫”。Chip-8 是一个非常简单的虚拟机，很多 70 年代的游戏都能在上面跑（比如 Pong, Tetris）。
**要做什么：**
你需要用 Go 模拟 CPU 的内存、寄存器和指令集（opcode）。读取一个二进制游戏 ROM 文件，解析指令（比如 `0x00E0` 是清屏），然后画在窗口上。

* **技术点：** 位运算 (`&`, `|`, `>>`), 十六进制处理, 简单的图形库（SDL2 或 Ebitengine）。
* **成就感：** 当你看到几百行代码真的跑起了几十年前的游戏，那种感觉无与伦比。

### 方向四：WebAssembly 游戏 —— 浏览器里的 Go

**口味：** 甜品（图形化、跨平台、WASM）
**描述：** Go 可以编译成 WebAssembly (WASM)，直接在浏览器里跑，不需要写一行 JavaScript。
**要做什么：**
使用 Go 的游戏引擎 `Ebitengine` (原 Ebiten) 写一个简单的 2D 游戏（比如像“Flappy Bird”或者“弹球”），然后编译成 `.wasm` 文件，放在网页上给朋友玩。

* **技术点：** `github.com/hajimehoshi/ebiten`, 游戏循环 (Update/Draw), 碰撞检测。
* **优势：** 既然不想做 TUI，那就做真正的 GUI，而且直接网页分发，逼格很高。

### 你想选哪个？

* **如果想实用且能解决日常痛点：** 选 **方向一（局域网闪传）**。
* **如果想深入理解网络底层：** 选 **方向二（简易代理）**。
* **如果想挑战硬核编程并了解计算机原理：** 选 **方向三（Chip-8 模拟器）**。
* **如果想做可视化的东西给别人玩：** 选 **方向四（WASM 游戏）**。
